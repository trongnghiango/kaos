## 4. What Was Decided (Do NOT Re-debate)

These decisions are final from the architecture planning session:

1. **Build a standalone `kaos scan` feature** — do NOT import the `knowledge-base` project. It's designed for natural language text (books, articles). Codebase scanning needs **AST-level accuracy** that only TypeScript Compiler API can provide.
2. **AST scan = rule-based (no LLM).** Only semantic enrichment (description, preconditions, exceptions) uses LLM.
3. **JSON storage** for knowledge graph, not Markdown. Machines query JSON faster.
4. **Git branch sandbox** for workspace isolation, not Docker. Lightweight, leverages existing git workflow.
5. **Remove the ban on `tsc`, `pnpm`, `node`** from agent prompts — agents need compilation feedback.
6. **Incremental updates via `git diff HEAD`** — only rescan changed files.
7. **Implementation order:** Domain → Bridge (AST parser) → Storage → Scanner Adapter → Use Case → CLI → Engine Integration → DI.
